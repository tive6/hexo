程序猿必学——Markdown语法

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过简单的标记语法，它可以使普通文本内容具有一定的格式，从而让文本易读易写，增强可读性。
因此，Markdown被广泛的应用于hexo个人博客、Github、Coding等具有大量文本的交流学习网站。

以下就是Markdown中常用的格式写法：

1、标题

  • 标题有六级，分为一级、二级…到五级、六级，类似于HTML中的h1-h6标签，标题文字前加#号，一级前边就是一个#，六级前边就是六个#,以此类推。

  • 提示：#与后边文字中间要加上一个空格。

md写法：

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
####### 六级标题

结果显示：

一级标题

二级标题

三级标题

四级标题

五级标题
六级标题

2、单行文本与多行文本

  • 单行文本与多行文本前加入两个Tab即可

    文本前加入两个Tab即可
文本前加入两个Tab即可

3、无序（ul）列表

md写法：

* 昵称：隔壁小王
* 别名：隔壁老王
* ID：小王

结果显示：

  • 昵称：隔壁小王
  • 别名：隔壁老王
  • ID：小王

4、有序（ol）列表

  • 一个数字+.+内容

md写法：

1. 前端
3. 后台
4. 测试
9. UI
6. 产品

结果显示：

  1. 前端
  2. 后台
  3. 测试
  4. UI
  5. 产品
  • 提示：这里的数字，并不需要按照一定顺序排列。

5、结构列表

md写法：

* 编程语言
    * 脚本语言
        * Python

结果显示：

  • 编程语言
    • 脚本语言
      • Python

6、一个带有title属性的图片（img标签）

md写法：

![面对疾风吧](http://upload-images.jianshu.io/upload_images/3876828-a4346506018aa44f.gif?imageMogr2/auto-orient/strip "哈赛给 啊痛")

结果显示：

7、一个超链接（a标签）

md写法：

[个人博客](http://zmnaer.com "zmnaer个人博客")

结果显示：

个人博客

8、一个有超链接的图片

md写法：

[![zmnaer]](http://zmnaer.com)
[zmnaer]:http://ohecg7vrp.bkt.clouddn.com/14.gif "blog"

结果显示：

9、换行符（br）

md写法：

第一行文字
<br/>
第二行文字

结果显示：

第一行文字

第二行文字

10、高亮文字、斜体、粗体、加粗斜体

md写法：

`高亮`显示的`文字`
*斜体*@文字
**粗体**@文字
***加粗斜体***@文字

结果显示：

高亮显示的文字
斜体@文字
粗体@文字
加粗斜体@文字

11、特别显示的文本块

md写法：

  • 三个·+bash(或者java、javascript、php、html…) +换行+代码块...+换行+三个·

结果显示：

  • 提示：此处的·是英文输入法下Tab键上边的那个字符，+号请忽略不要写。

12、表格

md写法：

|表头1|表头2|表头3|
|:---:|:---:|:---:|
|con11|con22|con33|
|con11|con22|con33|

结果显示：

表头1 表头2 表头3
con11 con22 con33
con11 con22 con33
  • 提示：此处的:相当于CSS中的text-align属性；:在左边就是文本左对齐；:在右边就是文本右对齐；如果两边都有，则本文居中显示(相当于text-align:center)。
  • 原文地址：http://zmnaer.com/tags/Markdown/

提示：

  • 一般Markdown文档都是以.md后缀结尾的；
  • Github、Coding等开源社区，在仓库的根目录下一般都存在README.md文件，它就是Markdown格式编写的，可以直接在仓库的下方展现出来，用于仓库的说明、或API的概述；
  • 文中所有的·都是英文输入法下Tab键上边的那个字符，+号请忽略不要写。（补充：高亮显示和特别显示中的·）；
  • Markdown支持大部分HTML标签，可以直接写HTML标签，并能加内联样式；
  • Markdown在各个地方解析的情况不同，因此结果显示会有所区别，有的平台不能解析表格。
  • 此文仅供参考，如有不详之处，望见谅。
